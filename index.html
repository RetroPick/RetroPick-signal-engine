Main Workflow

Dua mode kerja: cron MVP dan real-time trigger.

  1. Signal Engine berjalan setiap 5-10 menit menggunakan cron job.
  2. Service mengambil market terbaru dari endpoint RetroPick API.
  3. Service mengecek apakah market sudah pernah dikirim.
  4. Caption dibuat sesuai template komunitas.
  5. Caption dikirim ke Telegram dan Discord.
  6. Log pengiriman disimpan agar tidak spam atau duplikasi.
  1. Admin atau team publish market dari RetroPick.
  2. RetroPick API memanggil endpoint Signal Engine.
  3. Signal Engine memvalidasi secret token.
  4. Konten dikirim real-time ke Telegram dan Discord.
  5. Status delivery disimpan untuk monitoring.

  market_id TEXT NOT NULL,

Security

Credential hanya boleh hidup di server/service environment.

Jangan expose Telegram token, Discord webhook, atau broadcast secret ke client-side code.
Gunakan secret header untuk endpoint broadcast internal.
Batasi request agar tidak spam menggunakan rate limit.
Validasi marketId dan URL sebelum dikirim.
Log semua percobaan broadcast, baik berhasil maupun gagal.

Roadmap

Bangun kecil dulu, lalu tambah automasi.

Phase 1

MVP Broadcast Service

Setup repo, env, Telegram sender, Discord sender, manual endpoint.

Phase 2

Pull API Job

Cron latest market, check log, dan prevent duplicate.

Phase 3

Real-time Trigger

RetroPick API trigger /broadcast/market saat publish.

Phase 4

Growth Alerts

Trending, closing soon, dan daily recap untuk komunitas.

Phase 5

AI Caption

Generate caption dari market data agar lebih hemat waktu.

Acceptance Criteria

Checklist sebelum service dianggap siap MVP.

Final recommendation: mulai dari MVP manual dan pull API. Setelah stabil, lanjutkan real-time trigger, trending alert, daily recap, dan AI caption.
